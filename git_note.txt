
初始化仓库
git init

将文件添加至暂存区
git add <文件名>
添加多个文件的时候，. 点表示添加所有文件

将文件提交至仓库
git commit -m "本次提交说明" 文件名
提交多个文件时候，可以不加文件名称，默认是提交所有文件

查看当前仓库状态
git status

查看历史提交记录
git log   #展示所有版本的详细信息
git log --oneline  #以一行的方式展示版本信息
git reflog    #显示head信息，一般采用这种

回退版本
git reset --hard <版本id，一个16进制数>
hard 参数表示的是，仓库中的代码和暂存区和工作区中的代码都回到指定版本
可用于在删除工作区中的数据时，想恢复时，可直接回退到之前的状态
其他参数
--mixed
本地库的指针移动的同事，重置暂存区，但工作区不动
--soft  
本地库的指正移动的时候，bbcc，暂存区和工作区都不移动

diff 对比
git diff <文件名>
对比的是工作区和暂存区的文件内容有什么不同
git diff 
后面不加文件名，表示比对的是工作区所有文件
git diff HEAD <文件名>
将暂存区中的数据和仓库中的数据进行对比
git diff <索引id> <文件名>
和历史中的某一个版本对比

注意：git是以行为单位管理文件中的内容的

分支操作
git branch -v
查看所有分支
git branch <分支的名称>
在当前主分支的版本上创建新分支
git checkout <分支名称>
切换分支 
git merge <分支名称>
合并分支，将当前分支和目标分支进行合并
    注意：分支合并不一定成功，当两个分支在同一文件中的同一行进行修改，
    就会导致分支合并冲突，从而进入分支合并状态merging，这时候需要进行手工整合
    进去取舍，完毕后重新进行add 和commit（commit 操作时不能带文件名，否则报错） 操作即可完成分支合并

查看别名
git remote -v
起别名
git remote add <别名> <远程仓库别名>
起别名后，以后每次就可以通过别名来代替访问

推送操作
git push <别名或者地址> <分支名称>
将对应的分支推送到远程仓库中，入股远程仓库没有该分支，则会自动创建该分支

克隆操作
git clone <远程库的地址>
克隆操作可以帮助我们：
1、初始化本地仓库
2、将远程仓库的内容完全克隆岛本地
3、替我们创建远程库的别名

拉取操作
拉去操作 pull ，相当于 fetch + merge 操作
git fetch <别名或者地址> <分子名>
抓取操作，只是将远程库的内容抓取到本地库，但是工作区中的文件并未更新，还是原来的内容
git merge <远程库名称,例如 git_note/master>
git pull <别名或地址> <分支名称>
可以用pull 代替 fetch + merge 操作
