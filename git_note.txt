
初始化仓库
git init

将文件添加至暂存区
git add <文件名>
添加多个文件的时候，. 点表示添加所有文件

将文件提交至仓库
git commit -m "本次提交说明" 文件名
提交多个文件时候，可以不加文件名称，默认是提交所有文件

查看当前仓库状态
git status

查看历史提交记录
git log   #展示所有版本的详细信息
git log --oneline  #以一行的方式展示版本信息
git reflog    #显示head信息，一般采用这种

回退版本
git reset --hard <版本id，一个16进制数>
hard 参数表示的是，仓库中的代码和暂存区和工作区中的代码都回到指定版本
可用于在删除工作区中的数据时，想恢复时，可直接回退到之前的状态
其他参数
--mixed
本地库的指针移动的同事，重置暂存区，但工作区不动
--soft  
本地库的指正移动的时候，bbcc，暂存区和工作区都不移动

diff 对比
git diff <文件名>
对比的是工作区和暂存区的文件内容有什么不同
git diff 
后面不加文件名，表示比对的是工作区所有文件
git diff HEAD <文件名>
将暂存区中的数据和仓库中的数据进行对比
git diff <索引id> <文件名>
和历史中的某一个版本对比

注意：git是以行为单位管理文件中的内容的

分支操作
git branch -v
查看所有分支
git branch <分支的名称>
在当前主分支的版本上创建新分支
git checkout <分支名称>
切换分支
